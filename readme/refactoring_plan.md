
# Пошаговый план рефакторинга системы управления движением

## 1. Общая цель

Миграция системы управления 8 шаговыми двигателями с программной генерации импульсов (в прерывании таймера) на **аппаратную генерацию** с использованием ШИМ (PWM) на отдельных каналах таймеров. Это обеспечит стабильность, производительность и масштабируемость системы.

---

## 2. План работ по файлам

### **Этап 0: Конфигурация оборудования и генерация кода (STM32CubeMX)**

**Это обязательный предварительный шаг, который выполняется пользователем вручную.** Без него дальнейший рефакторинг невозможен.

**Файл:** `STM32F103_step_motors_refactored.ioc`

**Действия в STM32CubeMX (20.02.2026):**

1.  **Очистка GPIO:** Убедитесь, что все предыдущие назначения GPIO, относящиеся к управлению двигателями (STEP, DIR, ENABLE), удалены или освобождены.
2.  **Настройка Таймеров (TIM1 и TIM2):**
    *   Активируйте **TIM1** и **TIM2**.
    *   Для каждого из таймеров включите все четыре канала (Channel 1, 2, 3, 4) в режим **`PWM Generation CHx`**.
    *   **Параметры таймеров (вкладка `Parameter Settings`):**
        *   `Prescaler`: **63** (для тактовой частоты таймера 1 МГц)
        *   `Counter Period (ARR)`: **999** (для базовой частоты ШИМ 1 кГц)
        *   `Pulse` (для каждого канала): **500** (для скважности ~50%)
3.  **Настройка CAN (Автоматическое переназначение):**
    *   Активируйте CAN1. Убедитесь, что пины настроены следующим образом (CubeMX должен был их автоматически переназначить, если ранее были конфликты):
        *   `CAN_RX`: `PB8`
        *   `CAN_TX`: `PB9`
4.  **Настройка UART (для драйверов TMC2209):**
    *   Активируйте `USART1` и `USART2` в асинхронном режиме.
    *   **Финальная распиновка UART:**
        *   `UART1_RX`: `PB7`
        *   `UART1_TX`: `PB6`
        *   `UART2_RX`: `PA3`
        *   `UART2_TX`: `PA2`
5.  **Настройка GPIO (DIR и ENABLE):**
    *   Сконфигурируйте следующие пины как **`GPIO_Output`**. Для каждого пина рекомендуется установить `User Label` для ясности.
    *   **Пины направления (DIR):**
        *   `PB0` (DIR_1)
        *   `PB1` (DIR_2)
        *   `PB2` (DIR_3)
        *   `PB3` (DIR_4)
        *   `PB4` (DIR_5)
        *   `PB5` (DIR_6)
        *   `PB12` (DIR_7)
        *   `PB13` (DIR_8)
    *   **Пины включения (ENABLE):**
        *   `PA4` (EN_1)
        *   `PA5` (EN_2)
        *   `PA6` (EN_3)
        *   `PA7` (EN_4)
        *   `PA12` (EN_5)
        *   `PA15` (EN_6)
        *   `PB14` (EN_7)
        *   `PB15` (EN_8)
6.  **Проверка пинов STEP (Автоматическое назначение):** Убедитесь, что CubeMX автоматически назначил следующие пины для ШИМ-каналов:
    *   `TIM1_CH1`: `PA8`
    *   `TIM1_CH2`: `PA9`
    *   `TIM1_CH3`: `PA10`
    *   `TIM1_CH4`: `PA11`
    *   `TIM2_CH1`: `PA0`
    *   `TIM2_CH2`: `PA1`
    *   `TIM2_CH3`: `PB10`
    *   `TIM2_CH4`: `PB11`
7.  **Генерация кода:**
    *   После завершения всех настроек, выполните **`Project -> Generate Code`** (Ctrl+Shift+G). Это обновит проект, добавив в него код инициализации HAL для сконфигурированной периферии.
