
# Предложение по рефакторингу системы управления движением

**Дата:** 20.02.2026
**Автор:** Gemini AI Assistant
**Статус:** Предложение

## 1. Введение и цель

Данный документ описывает техническое обоснование и план рефакторинга подсистемы управления шаговыми двигателями в проекте.

Целью рефакторинга является переход на новую архитектуру, способную обеспечить стабильную, быструю и масштабируемую работу **8 шаговых двигателей** на аппаратной платформе **STM32F103**. Текущая реализация, хотя и является функциональной для малого числа осей, несет в себе фундаментальные риски при масштабировании до требуемого уровня.

## 2. Анализ текущей архитектуры (Метод "Один таймер + Прерывания")

Текущая система генерирует управляющие STEP-импульсы программно внутри обработчика прерывания таймера (`TIM2_IRQHandler`).

*   **Принцип работы:** Таймер используется как высокоточный "будильник", который вызывает прерывание для каждого шага. Логика внутри прерывания переключает GPIO-пин и вычисляет время до следующего шага.
*   **Преимущества:** Высокая гибкость в реализации сложных профилей движения (ускорения/замедления) программными методами.
*   **Критические недостатки для 8 двигателей:**
    *   **Высокая нагрузка на CPU:** Каждый шаг каждого мотора вызывает прерывание. Для 8 моторов на скорости 2000 шаг/с это создает нагрузку в 16000 прерываний в секунду, отнимая у процессора ресурсы на основную логику.
    *   **Уязвимость к джиттеру (Jitter):** Исполнение прерывания может быть задержано другими системными событиями (например, CAN, UART), что приводит к неравномерности импульсов, потере шагов и срыву позиционирования на высоких скоростях.
    *   **Низкая масштабируемость:** Производительность и стабильность системы деградируют с увеличением количества одновременно движущихся моторов. Система в ее текущем виде не может гарантировать надежную работу 8 осей.
    *   **Обнаруженные дефекты:** В ходе анализа были выявлены критические ошибки (жестко заданный ID мотора, блокировка задачи после движения), что указывает на сложность и хрупкость текущего подхода.

## 3. Предлагаемая архитектура (Метод "Аппаратная генерация импульсов")

Предлагается перейти на архитектуру, где генерация STEP-импульсов полностью переложена на аппаратные ресурсы микроконтроллера.

*   **Принцип работы:** На каждый из 8 двигателей выделяется отдельный канал таймера (например, каналы таймеров TIM1, TIM2, TIM3, TIM4). Таймер настраивается на генерацию точного числа импульсов (в режиме PWM или Output Compare) напрямую на STEP-пин, сконфигурированный в режиме альтернативной функции (AFIO).
*   **Преимущества:**
    *   **Нулевая нагрузка на CPU во время движения:** Процессор только запускает, останавливает или изменяет параметры движения. Сама генерация импульсов происходит аппаратно.
    *   **Аппаратная точность:** Полное отсутствие джиттера. Стабильность импульсов гарантируется тактовой частотой периферии и не зависит от программной логики.
    *   **Высокая производительность:** Максимальная скорость движения ограничена только возможностями драйвера и мотора, а не производительностью CPU.
    *   **Надежность и масштабируемость:** Нагрузка на систему не зависит от количества активных моторов.

## 4. Сравнительная таблица подходов (для 8 моторов на STM32F103)

| Характеристика | Текущий подход (Прерывания) | Предлагаемый подход (Аппаратный) |
| :--- | :--- | :--- |
| **Ресурсы таймеров** | 1 таймер | 8 каналов (например, TIM1+TIM2) |
| **Нагрузка на CPU** | **Высокая**, растет с числом моторов и скоростью | **Нулевая** во время движения |
| **Стабильность** | **Низкая**, уязвима к прерываниям | **Идеальная**, гарантирована аппаратно |
| **Макс. скорость** | **Ограничена** производительностью CPU | **Ограничена** физикой мотора/драйвера |
| **Надежность** | **Низкая** при масштабировании | **Высокая** и предсказуемая |
| **Масштабируемость**| **Плохая** | **Отличная** |

## 5. Обоснование для рефакторинга

Текущая архитектура не соответствует требованиям проекта по управлению 8 двигателями. Попытки ее оптимизации приведут к незначительным улучшениям, но не решат фундаментальную проблему узкого места в виде программной генерации импульсов в прерывании.

Переход на аппаратную генерацию импульсов (Подход 2) является **стратегически необходимым** шагом для обеспечения надежности, производительности и предсказуемости системы. Микроконтроллер STM32F103 обладает всеми необходимыми аппаратными ресурсами для реализации данного подхода. Этот рефакторинг позволит создать прочный фундамент для дальнейшей разработки сложной кинематики и алгоритмов управления.
